Fresca JS Library

All of our projects are different, all of our JavaScript is different. Should we not instead build a new standard set of plugins into Blue Print? Essentially these will be locked files that can't be amended during a project.

If any amendments/improvements are required to a global plugin for a specific client, then they can be made using the jQuery extend functionality to override specific methods. Writing all our code as jQuery plugins will provide the encapsulation and reusability that is frequently missing from our code.

Usage

For each plugin, we should be able to add/override methods without having to create a new plugin. The jQuery widget factory provides a really nice, compact way to do this. As an example, for an existing method, we can just do this:

Load in the library

<script src="fresca-library.js"></script>

Call the library

Very often we are modifying mark up, from this we cannot determine what JavaScript (if any) is attached. Would using data attributes be preferable to id's and class names? We could even namespace these data attributes like so:

<a href="#" data-reveal-events="click" data-reveal-target="#myID">Open modal window</a>

Extend the library

The idea is that a developer could add/override individual methods without touching the shared file. This would be really useful for setting which JS templating engine to use or specifying dependencies on other libraries etc.

$.widget("fresca.zoom", {
	options: {
		// Default options
	}, somePrivateMethod: function(argument) {
		// Create a private method
	}, somePublicMethod: function(argument) {
		// Create a public method
	}
});
$('#my-zoom-image').zoom({
	'_somePrivateMethod': 'someVar',
	'somePublicMethod': 'someVar'
});
